# GitHub Workflow

You are running as an automated agent. Work autonomously without asking the user questions.

## Step 1: Find unassigned issues

Run these commands to find open issues WITHOUT the `claude-wip` label:

```bash
gh issue list --repo Komediruzecki/finance-manager --state open --json number,title,body,labels --limit 20
```

Filter out any issue that already has the `claude-wip` label. If no unhandled issues remain, say `No new issues` and stop.

## Step 2: Pick ONE issue (oldest first) and claim it

```bash
gh issue edit NUMBER --add-label claudewip --repo Komediruzecki/finance-manager
```

## Step 3: Understand the issue

Read the issue body. Determine what needs changes.

Read all relevant files before planning.

## Step 4: Create a branch and implement

```bash
git checkout main && git pull
git checkout -b feat/issue-NUMBER-short-description
```

Plan the implementation, then make the changes. Follow these rules strictly:
- Use Conventional Commits: feat:, fix:, refactor:, etc.
- Never push to main — always create a PR
- Read files before editing them
- Be careful not to introduce security vulnerabilities
- Keep changes minimal and focused on the issue

## Step 5: Commit, push, and create a PR

Stage only the files you changed. Commit with a descriptive message. Push and create a PR:

```bash
gh pr create --repo Komediruzecki/finance-manager --title \"type: short description\" --body \"...\"
```

Include `Closes #NUMBER` in the PR body.
If the issue spans BOTH repos, create a PR in each repo and cross-reference them.

## Step 6: Continue to next issue — do NOT stop after PR creation
